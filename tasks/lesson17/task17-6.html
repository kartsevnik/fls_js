Class: Task 1

Дано два класи MultChecker (клас для перевірки таблиці множення - рандомно генеруються числа, які треба перемножати), AddChecker (клас для перевірки додавання - рандомно генеруються числа у заданому діапазоні, які треба додавати). Обидва класи надсилають результати тестування об'єкту класу Hystory, який зберігає історію тестування у масиві у вигляді об'єктів

Приклад.

testsList= [

{firstNum:1, secondNum:5,opration:’*’, userAnswer:7, correctAnswer:5},

{firstNum:3, secondNum:4,opration:’+’, userAnswer:7, correctAnswer:7},

]

Можна створити окремий клас TestData, який описує один такий тест і у якому будуть ці поля. Розробити клас TestManager, який використовуючи ці класи за допомогою таймера періодично генерує якісь N задач (рандомно вибираємо, що опитувати: додавання чи множення) і проводить опитування. Результати тестування додаються в об’єкт History. Зробити так, щоб об'єкт такого класу можна було створити тільки один. Коли зроблено ці N задач вивести усю історію на екран.

Two Multchecker classes are given (a class for checking the multiplication table - randomly generated numbers that need to be multiplied), Addchecker (class to check the addition - randomly generated numbers in a given range to be added).Both classes send the test results of the Hystory class that stores the history of testing in the array in the form of objects

Example.

testslist = [

{Firstnum: 1, Secondnum: 5, Opposition: '*', Useranswer: 7, Correctanswer: 5},

{Firstnum: 3, Secondnum: 4, Opposition: '+', Useranswer: 7, Correctanswer: 7},

]

You can create a separate Testdata class that describes one such test and in which these fields will be. Develop a TestManager class that, using these classes with timer, periodically generates some N tasks (randomly select what to survey: addition or multiplication) and conducts a survey.Test results are added to the History.Make sure that the object of this class can be created only one.When these n tasks are done to bring the whole story on the screen.
